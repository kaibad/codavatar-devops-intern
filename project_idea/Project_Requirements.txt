# DevOps Project Requirements

1. Blue-Green Deployment in Kubernetes

2. Secure CI/CD Pipeline

3. Use self-managed Kubernetes (MicroK8s preferred over EKS)

4. Frontend Deployment:

   * Amazon S3

5. Backend Deployment:

   * EC2

6. Database:

   * PostgreSQL/MySQL on EC2 OR
   * Amazon RDS
   * Database Backups

7. Implement Least Privilege Access

8. End-to-End Automation

9. Kubernetes Best Practices:

   * Resource Requests and Limits
   * ConfigMaps
   * Secrets
   * HashiCorp Vault / AWS Secrets Manager
   * Cluster Monitoring
   * Logging
   * Observability
   * Namespace Separation (dev, staging, prod)
   * Persistent Volumes (PV)
   * Persistent Volume Claims (PVC)
   * Rolling Update Strategy
   * Blue-Green Deployment Strategy
   * Non-Root Containers
   * Read-Only Filesystem (where applicable)

10. AIOps for Monitoring and Alerting

11. Auto Scaling:

* Scale up to 5 Pods
* Scale up to 5 Nodes
* Alert via Slack and/or Email if scaling exceeds threshold

12. Security:

* RBAC
* IAM Roles
* Service Accounts
* GitHub OIDC Authentication
* Container Image Scanning (Trivy)
* Base Image Hardening

13. Ingress Controller

14. Kubernetes Tools:

* Helm
* ArgoCD

15. Kubernetes Health Checks:

* Liveness Probe
* Readiness Probe
* Startup Probe

16. Terraform for Infrastructure Provisioning:

* VPC
* Public Subnets
* Private Subnets
* Route Tables
* Security Groups
* EC2
* S3
* RDS (Optional)
* Bastion Host (or VPN Access)
* Kubernetes resources managed manually

17. GitHub as Single Source of Truth:

* Application Code
* Terraform Code
* Helm Charts
* Dockerfiles
* Kubernetes Manifests
* CI/CD Pipelines
* Documentation

18. Terraform Remote Backend:

* S3 Backend
* DynamoDB State Locking

19. Terraform CI/CD Pipeline:

* Pull Request Validation Pipeline
* Branch Protection Rules
* terraform fmt
* terraform validate
* Security Scan (Checkov/tfsec)
* terraform plan
* Manual Approval (Production)
* terraform apply
* Update Remote State

20. Environment Strategy:

* Development
* Staging
* Production

21. Terraform State File Security:

* No Secrets in State File
* Encrypt Backend
* Restricted Access

22. Infrastructure Automation:

* No nedd to 100% automate the infra but 
* Secure Credential Management
* No Hardcoded Secrets
* IAM Roles / GitHub OIDC Authentication

23. Storage & Recovery:

* S3 Versioning
* Database Backup Strategy

24. Documentation:

* Architecture Diagram
* CI/CD Flow Diagram
* Deployment Guide
* Troubleshooting Guide
* Runbook
