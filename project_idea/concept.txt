The project: VoiceDesk — AI-powered customer support platform

The idea that ties everything together: a company runs a customer support product where customers call in (VoIP via Twilio) or chat (AI chatbot), and every interaction gets queued through SQS, processed by a Python API, stored in PostgreSQL, and displayed on a React dashboard. This gives a real, coherent reason for every technology on your list.


Month 1 — Foundation (weeks 1–4)

Week 1 — App + containers
Build the 3-tier app: React frontend, Python/Flask REST API, PostgreSQL. Dockerize all three with a docker-compose.yml for local dev. Add an Nginx container as reverse proxy and load balancer. Write bash scripts for setup.sh, deploy.sh, healthcheck.sh.

Week 2 — CI/CD pipeline
Set up Jenkins with a Jenkinsfile in the repo. Pipeline stages: lint → unit test → build Docker image → push to ECR → deploy to dev. Add SQS integration to the Python API (async job processing). Wire up Twilio webhooks for VoIP call events. Add a continuous testing strategy: pytest for unit tests, Postman/Newman for API integration tests, k6 for load testing.

Week 3 — Terraform infrastructure
Write Terraform to provision: VPC, EC2 or EKS nodes, RDS PostgreSQL, ECR repo, SQS queue, IAM roles. Use S3 + DynamoDB for remote state and locking. Separate dev.tfvars / prod.tfvars for multi-env. Vault for secret injection (DB passwords, API keys, Twilio credentials).

Week 4 — MicroK8s + Helm
Install MicroK8s locally or on a VM. Convert your Docker Compose into Helm charts with values-dev.yaml, values-staging.yaml, values-prod.yaml. Add HPA for the Python API (scale on CPU > 60%). Deploy Nginx ingress controller. Set up ArgoCD and point it at your GitHub repo — now every git push deploys automatically (GitOps workflow).


Month 2 — Advanced features (weeks 5–8)

Week 5 — Blue-green deployment + HA
Implement blue-green using two Helm releases (voicedesk-blue, voicedesk-green) and flipping the Nginx ingress service selector. Write a bash script that automates the traffic switch and rollback. Add pod disruption budgets and multi-replica deployments for high availability. Demonstrate autoscaling with a k6 load test.

Week 6 — Monitoring + Slack alerts
Deploy Prometheus + Grafana via Helm (kube-prometheus-stack). Build a Grafana dashboard showing: request rate, error rate, latency (p95/p99), queue depth (SQS), pod count. Configure AlertManager to send Slack alerts on: pod crash loops, high error rate, SQS backlog > 100 messages, deployment anomalies.

Week 7 — AIOps + AI chatbot
AIOps: write a Python script that queries Prometheus metrics via its API, runs a simple anomaly detection (Z-score or moving average), and posts a Slack alert when something looks abnormal — without a human writing the threshold rule. Chatbot: build a /chat endpoint in your Flask API that calls an LLM (Claude API or OpenAI) with a system prompt describing your product. Wire the React frontend to show the chat widget to customers.

Week 8 — Git branching strategy, docs, demo
Enforce Git flow: main (prod-only via PR), staging, develop, feature/*, hotfix/*. Add branch protection rules. Write CONTRIBUTING.md and a full README.md with architecture diagram. Record a demo video or live walkthrough: commit → CI pipeline → ArgoCD sync → blue-green switch → Grafana shows the spike → Slack alert fires → chatbot answers a question.
